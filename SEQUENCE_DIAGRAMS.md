## Notes

### Timing Characteristics
- **Bot creation**: 2-10s (depends on image pull)
- **Bot start/stop**: 1-3s (container lifecycle)
- **Status update latency**: 0-5s (polling interval)
- **Event-driven updates**: <100ms (immediate WebSocket broadcast)
- **WebSocket reconnection**: 3s delay

### Error Handling
- All async operations have try/catch blocks
- Docker errors emit 'bot:error' events
- Frontend shows error alerts on operation failures
- WebSocket auto-reconnects on disconnect

### Concurrency
- Multiple clients can connect simultaneously
- All clients receive same broadcast messages
- REST operations are synchronous (one at a time per bot)
- Status polling runs independently in background
